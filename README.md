Персистентна (постійна) кумулятивна вікі-система, яка використовує агентів на базі LLM для імпорту, структурування, перехресного зв'язування та підтримки бази знань на основі нових джерел. Проєкт слугує шаблоном для створення персональних (або командних) баз знань під керуванням LLM із підтримкою лінтингу, запитів та каскадних оновлень.

## Структура

- [/AGENTS.md](AGENTS.md) - Основні правила поведінки агентів. Рекомендується не вносити в цей файл жодних змін без чіткого розуміння, навіщо ви це робите.
- [/.agents/](.agents/) — Автоматизовані робочі процеси агентів (навички `ingest`, `query`, `linter`).
- [/templates/](/templates/) — Шаблони для створення сторінок, джерел тощо.
- [/inbox/](inbox/) — Папка для нових необроблених матеріалів та медіафайлів (вхідна черга).
- [/raw/](raw/) — Незмінні вихідні джерела інформації.
- [/wiki/](wiki/) — Основна база знань (концепції, сутності, архіви).
  - [index.md](wiki/index.md) — Покажчик (індекс) усіх сторінок.
  - [log.md](wiki/log.md) — Журнал (лог) активності та операцій.

## Інструкція користувача

### Клонування репозиторію

Клонуйте цей репозиторій на свій комп'ютер:

```bash
git clone https://github.com/BogdanovychA/llm-wiki
```

> [!NOTE]
> В залежності від обраного агента, необхідно скопіювати (перейменувати) [AGENTS.md](AGENTS.md) у файл із відповідним ім'ям, який цей агент читає при старті (наприклад, `CLAUDE.md` для Claude Code), або створити символічне посилання. Уточніть це в документації до вашого агента.

### Додавання джерел

Якщо у вас є нові матеріали для наповнення бази знань:
1. Помістіть **текстові файли** (`.md`, `.txt` тощо) у каталог [inbox/](inbox/).
2. При наявності **вкладень та медіафайлів** (на які посилаються текстові джерела), помістіть їх у каталог [inbox/assets/](inbox/assets/).

### Робота з агентом та команди

Для взаємодії з вікі використовуйте LLM-агентів (наприклад: [OpenCode](https://opencode.ai/), [Antigravity](https://antigravity.google), [Claude Code](https://claude.com/product/claude-code) тощо). 

Залежно від завдання, давайте агенту такі вказівки:
- **Імпортувати джерела (`ingest`)** *(виконується після їх додавання)*:
  *«Оброби нові матеріали»*, *«Імпортуй джерела»* тощо. Агент автоматично структурує інформацію, створить/оновить сторінки концепцій або сутностей, перенесе першоджерела в незмінний архів та оновить покажчики.

- **Пошук та аналіз (`query`)**:
  *«Знайди інформацію про...»*, *«Порівняй концепції... та...»*, *«Що ти знаєш про...?»* тощо. Агент знайде потрібне та за необхідності збереже складну відповідь в архів запитів.

- **Перевірка цілісності (`lint`)**:
  *«Перевір вікі»*, *«Зроби лінтинг»* тощо. Агент перевірить базу знань на наявність битих посилань чи логічних суперечностей. Рекомендується періодично здійснювати таку перевірку.

### Перегляд бази знань

Для перегляду та зручної навігації по вікі ви можете використовувати будь-який редактор Markdown. Рекомендуються застосунки на кшталт [Obsidian](https://obsidian.md), які дозволяють візуалізувати зв'язки між сторінками у вигляді графа та спрощують перехід за відносними посиланнями.
