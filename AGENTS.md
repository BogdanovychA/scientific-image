# Інструкції та конвенції для розробників і AI-агентів

Цей документ визначає архітектуру проєкту, правила розробки, обмеження на редагування та критичні алгоритми збірки сайту «Науковий образ світу».

---

## 1. Структура проєкту

Проєкт базується на генераторі статичних сайтів **MkDocs** з темою **Material for MkDocs**. Основні каталоги та файли:

```
.
├── AGENTS.md               # Цей файл (інструкції для розробників та системних агентів)
├── src/
    └── hooks.py            # Python-хуки збірки MkDocs (динамічна генерація меню, редіректів)
├── mkdocs.yml              # Основний конфігураційний файл сайту
├── pyproject.toml          # Залежності та конфігурація розробки (Ruff, ruff-format)
├── uv.lock                 # Lock-файл пакета менеджера uv
├── overrides/              # Кастомні перевизначення шаблонів теми Material
│   └── main.html           # Шаблон розмітки відео та кнопки «На головну»
└── data/                   # Каталог із вихідними даними (наповнення сайту)
    ├── AGENTS.md           # Інструкції з імпорту та опрацювання контенту вікі
    ├── raw/                # Необроблені первинні джерела
    ├── wiki/               # Вікі-база знань (концепції, сутності, архів)
    ├── scripts/            # Клієнтські JavaScript-файли (блок випадкових статей)
    └── stylesheets/        # Кастомні CSS-стилі (вікі-стиль Вікіпедії, вбудоване відео)
```

---

## 2. Критичні конвенції та правила розробки

### 2.1. Сувора заборона на ручне редагування каталогу `data/`
* **Правило:** AI-агентам та розробникам заборонено вручну створювати або переміщувати файли у корені каталогу `data` або напряму змінювати структуру контенту.
* **Виняток:** Дозволяється створювати й редагувати лише стилі у `data/stylesheets/` та клієнтські скрипти у `data/scripts/`.
* **Чому це важливо:** Вся філософія сайту зводиться до автоматизації. Будь-які структурні зміни, генерація покажчиків, створення стартових редіректів тощо мають виконуватися виключно через програмні хуки збірки у `src/hooks.py`.
* **Контент та джерела:** Операції з наповнення вікі, імпорту джерел та оновлення покажчиків виконуються згідно з окремим регламентом, що детально описаний у файлі `data/AGENTS.md`.

### 2.2. Заборона статичної навігації (`nav:`) у `mkdocs.yml`
* **Правило:** У файлі `mkdocs.yml` заборонено фіксувати список сторінок через блок `nav:`.
* **Як це працює:** Меню навігації генерується **динамічно** під час кожної збірки за допомогою хука `on_config(config)` у `src/hooks.py`. Він автоматично сканує директорію `data` і розподіляє сторінки на розділи: «Головна» (`wiki/index.md`), «Статті» (`wiki/concepts/`, `wiki/entities/`, `wiki/archives/`) та «Джерела» (`raw/`).
* **Результат:** Додавання нових статей чи джерел у папку `data` не потребує жодних змін у файлах конфігурації.

### 2.3. Генерація редіректу на головну сторінку
* **Правило:** Заборонено створювати файл `data/index.md` вручну для створення головної сторінки.
* **Як це працює:** Оскільки основна сторінка вікі за структурою розташована у `wiki/index.md` (компілюється у `build/wiki/index.html`), для коректного завантаження сайту з кореня (за адресою `/`) хук `on_post_build(config)` у `src/hooks.py` автоматично створює файл `index.html` у вихідній директорії. Цей файл містить миттєвий клієнтський редірект на `wiki/`.

### 2.4. Сортування української кирилиці незалежно від локалі
* **Правило:** Для сортування алфавітного покажчика в `src/hooks.py` заборонено використовувати системні локалі через `locale.setlocale`.
* **Чому це важливо:** В середовищах CI/CD (GitHub Actions, Docker тощо) українська локаль `uk_UA.UTF-8` зазвичай відсутня, що призводить до збоїв. Стандартне сортування Python сортує символи за кодами Unicode, через що літери `Є`, `І`, `Ї` помилково опиняються вище за `А`, а `Ґ` — після `Я`.
* **Рішення:** Використовувати чистий Python-словник відповідності символів українського алфавіту (`UKRAINIAN_SORT_MAP`). Пріоритет сортування: спочатку українські літери, потім латинські, в кінці — цифри та спеціальні символи (наприклад, лапки `«`).

### 2.5. Фільтрація статей у клієнтському JS (блок «Випадкові статті»)
* **Правило:** При визначенні того, чи є посилання статтею у `data/scripts/random-articles.js`, заборонено перевіряти наявність слешу наприкінці URL (`/`).
* **Чому це важливо:** Якщо сайт збирається для локального/офлайн перегляду, де посилання закінчуються на `.html`, перевірка слешу зламається, і блок випадкових статей буде порожнім.
* **Рішення:** Використовувати повний шлях `a.href`, розрахований браузером, та перевіряти його на приналежність до папок категорій: `concepts/`, `entities/` або `archives/` (виключаючи `raw/` та `index`).
